# 启动方式

## 1. 项目说明

当前项目是一个基于 `FastAPI` 的服务，API 入口在：

- `src/apps/api/app.py`
- ASGI 应用对象：`src.apps.api.app:app`

项目默认使用 SQLite，本地无需先安装 PostgreSQL。PostgreSQL 仍可作为可选数据库通过配置启用。

## 2. 启动前准备

### 2.1 安装依赖

在项目根目录执行：

```powershell
pip install -r requirements.txt
```

如果你使用仓库内已有虚拟环境：

```powershell
.\venv\Scripts\python.exe -m pip install -r requirements.txt
```

### 2.2 数据库配置

数据库配置优先级：

1. 环境变量 `DATABASE_URL`
2. `.env` 中的 `DATABASE_URL`
3. `task_forge_config.json` 中的 `database.url`
4. 默认值 `sqlite:///data/task_forge.sqlite3`

默认情况下不需要配置任何环境变量，项目会使用：

```text
sqlite:///data/task_forge.sqlite3
```

如果要显式使用配置文件，可在项目根目录创建 `task_forge_config.json`：

```json
{
  "database": {
    "driver": "sqlite",
    "url": "sqlite:///data/task_forge.sqlite3"
  }
}
```

如果要使用 PostgreSQL，可设置环境变量：

```powershell
$env:DATABASE_URL = "postgresql+psycopg://postgres:ayu0v0@127.0.0.1:5432/postgres"
```

也可以写入 `.env`：

```env
DATABASE_URL=postgresql+psycopg://postgres:ayu0v0@127.0.0.1:5432/postgres
```

## 3. 初始化数据库

第一次启动前执行迁移：

```powershell
.\venv\Scripts\python.exe -m alembic upgrade head
```

也可以使用通用建表脚本创建 SQLAlchemy metadata 中的表：

```powershell
.\venv\Scripts\python.exe .\scripts\create_db_tables.py
```

推荐优先使用 Alembic，保证迁移版本可追踪。

## 4. 启动 API 服务

在项目根目录执行：

```powershell
.\venv\Scripts\python.exe -m uvicorn src.apps.api.app:app --host 127.0.0.1 --port 8000 --reload
```

## 5. 启动 Worker

```powershell
.\venv\Scripts\python.exe -m src.apps.worker.main
```

API 和 Worker 使用同一套数据库配置解析逻辑。不要给两者配置不同的数据库 URL，除非你明确需要隔离环境。

## 6. 启动后可访问的地址

- 健康检查：`http://127.0.0.1:8000/health`
- Batch 控制台：`http://127.0.0.1:8000/console/batches`
- Agent 控制台：`http://127.0.0.1:8000/console/agents`

## 7. 常见本地验证命令

默认 SQLite 测试：

```powershell
Remove-Item Env:DATABASE_URL -ErrorAction SilentlyContinue
.\venv\Scripts\python.exe -m pytest
```

PostgreSQL 回归测试：

```powershell
$env:DATABASE_URL = "postgresql+psycopg://postgres:ayu0v0@127.0.0.1:5432/postgres"
F:\python3.12.0\python.exe -m pytest
```

本地数据库流转验证：

```powershell
.\venv\Scripts\python.exe .\scripts\verify_db_flow.py
```

## 8. 最小启动流程

```powershell
cd L:\Project\task-forge
Remove-Item Env:DATABASE_URL -ErrorAction SilentlyContinue
.\venv\Scripts\python.exe -m alembic upgrade head
.\venv\Scripts\python.exe -m uvicorn src.apps.api.app:app --host 127.0.0.1 --port 8000 --reload
```

启动成功后，可用下面命令确认服务正常：

```powershell
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8000/health
```

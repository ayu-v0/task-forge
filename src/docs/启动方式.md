# 启动方式

## 1. 项目说明

当前项目是一个基于 `FastAPI` 的服务，API 入口在：

- `src/apps/api/app.py`
- ASGI 应用对象：`src.apps.api.app:app`

项目默认使用 SQLite，本地无需先安装 PostgreSQL。推荐通过 `cmd/start.ps1` 启动项目。

## 2. 启动前准备

### 2.1 安装依赖

推荐直接让启动脚本安装依赖：

```powershell
.\cmd\start.ps1 -InstallDeps
```

也可以手动安装：

```powershell
.\venv\Scripts\python.exe -m pip install -r requirements.txt
```

如果没有 `.venv`，脚本会回退使用系统 `python`。

### 2.2 数据库配置

数据库配置优先级：

1. 环境变量 `DATABASE_URL`
2. `.env` 中的 `DATABASE_URL`
3. `task_forge_config.json` 中的 `database.url`
4. 默认值 `sqlite:///data/task_forge.sqlite3`

仓库根目录已提供默认配置：

```json
{
  "database": {
    "driver": "sqlite",
    "url": "sqlite:///data/task_forge.sqlite3"
  }
}
```

默认 SQLite 数据库文件会写入：

```text
data/task_forge.sqlite3
```

如果要使用 PostgreSQL，可设置环境变量：

```powershell
$env:DATABASE_URL = "postgresql+psycopg://postgres:ayu0v0@127.0.0.1:5432/postgres"
```

也可以写入 `.env`：

```env
DATABASE_URL=postgresql+psycopg://postgres:ayu0v0@127.0.0.1:5432/postgres
```

## 3. 使用 start.ps1 启动

最小启动：

```powershell
.\cmd\start.ps1
```

脚本会执行：

- 解析当前数据库 URL 并打印。
- 如果当前数据库是 SQLite，自动执行 `alembic upgrade head`。
- 启动 API 服务。
- 写入日志和 PID 到 `.run/`。
- 启动后访问 `/health` 做健康检查。

启动 API 和 Worker：

```powershell
.\cmd\start.ps1 -WithWorker
```

指定 API 端口：

```powershell
.\cmd\start.ps1 -ApiPort 8010
```

端口被占用时，脚本默认会向后寻找可用端口。如果希望端口被占用时直接失败：

```powershell
.\cmd\start.ps1 -NoPortFallback
```

强制执行数据库迁移：

```powershell
.\cmd\start.ps1 -Migrate
```

跳过数据库迁移：

```powershell
.\cmd\start.ps1 -SkipMigrate
```

构建前端资源后启动：

```powershell
.\cmd\start.ps1 -BuildWeb
```

安装依赖、构建前端、启动 API 和 Worker：

```powershell
.\cmd\start.ps1 -InstallDeps -BuildWeb -WithWorker
```

## 4. 启动后可访问的地址

默认端口是 `8000`：

- 健康检查：`http://127.0.0.1:8000/health`
- Agent 控制台：`http://127.0.0.1:8000/console/agents`
- Batch 控制台：`http://127.0.0.1:8000/console/batches`

如果端口自动 fallback，请以脚本输出的 URL 为准。

## 5. 日志位置

启动脚本会把运行文件写到 `.run/`：

- API stdout：`.run/api.<port>.out.log`
- API stderr：`.run/api.<port>.err.log`
- API PID：`.run/api.pid`
- Worker stdout：`.run/worker.out.log`
- Worker stderr：`.run/worker.err.log`
- Worker PID：`.run/worker.pid`

## 6. 手动命令

一般不需要手动执行以下命令；排查问题时可以使用。

手动执行迁移：

```powershell
.\venv\Scripts\python.exe -m alembic upgrade head
```

手动启动 API：

```powershell
.\venv\Scripts\python.exe -m uvicorn src.apps.api.app:app --host 127.0.0.1 --port 8000
```

手动启动 Worker：

```powershell
.\venv\Scripts\python.exe -m src.apps.worker.main
```

本地数据库流转验证：

```powershell
.\venv\Scripts\python.exe .\scripts\verify_db_flow.py
```

## 7. 测试命令

默认 SQLite 测试：

```powershell
Remove-Item Env:DATABASE_URL -ErrorAction SilentlyContinue
.\venv\Scripts\python.exe -m pytest
```

PostgreSQL 回归测试：

```powershell
$env:DATABASE_URL = "postgresql+psycopg://postgres:ayu0v0@127.0.0.1:5432/postgres"
F:\python3.12.0\python.exe -m pytest
```
